无知的力量：AI 编程实现 Hexo 博客样式优化

2025-01-20

我浅学过一些编程语言，大概是能复制网上代码修改到完成课程作业的水平，但对于前端知识茫然无知，并且苦于网络信息的时效性，许多想法没能实现，只好搁置。至于 AI 编程，之前只是在对话框里向 AI 讨要过代码，没奢望过什么别的。这一回是在社交媒体的熏陶之下知道了 Cursor，恰巧又有点时间，所以试上一试。

上次听说 Cursor 还是号称可以免费用 GPT-4，现在则是配合 Claude-3.5-Sonnet 做自然语言编程。我试着给博客加了两个小部件，一个是目录，一个是点赞。

想起之前为了渲染 \(\LaTeX\) 公式而折腾的经历，主题不配合就算了，网上的教程版本多半也都对不上，好在最后找到一篇合适的文章解决了问题。

这种经历多了，人确实会丧失折腾的欲望——有这时间还不如干点正事。

今天有比较完整的时间，就下载 Cursor 试试。界面和 VSCode 基本一致，左边栏是文件夹、扩展插件之类的，右边栏是 AI 功能：Chat、Composer 和 Bug Finder。体验下来主要用到前两个，其中 Chat 是聊天启发思路用的，Composer 就是用来写代码的。为了节约限量次数，我把 Chat 模型设为 GPT-4o-mini，Composer 模型还是 Claude-3.5-Sonnet，能保证代码质量。

我打开之前的主题文件夹"Hacker"，把 Composer 的模式从 Normal 改为 Agent，然后就是详细提需求，一轮轮修改的过程：

  1. Composer 给出代码，我先保存一下（好像默认也会保存），然后在命令行敲 hexo s，用浏览器看看效果；
  2. 满意就选 Accept all；
  3. 不满意就选 Reject all，继续描述需求，循环往复。

可能是第一次用，感觉没省时间。因为一旦开始折腾，想法和要求就会越来越多，之前“算了再说”的问题，现在有希望一并解决了。还会遇到之前压根没想过的细节问题：鼠标和手指是不一样的，PC 端的设计放在手机上就很怪。比如，手机端收起的目录白边影响视野，点赞后按钮颜色不能立即变化，得先点击别处才行。

为了方便实时预览修改效果，在 Chrome 浏览器按 F12 再按 Shift+Ctrl+M 可以切换 PC 和移动端界面。解决问题之后，退出 Cursor 解除文件占用，命令行输入 hexo clean，再 hexo g -d，稍等一会儿就改好了。

确实是"黑盒"，而且写博客的代码我是真不懂，也不知道该写什么了，就看下效果吧：

总之，自然语言编程在现阶段给我最大的启发，一是把话讲清楚，二是不妨先试试。

25/01/16

几天之后，补充一下阶段性成果，我把项目发到 Github 上了：

除了目录和点赞之外，比较明显的修改是代码块样式和移动端的夜间模式，不太明显的有对各页面的微调和对 Cursor 改坏了但我没及时发现的部分的补救。作为 Cursor 编程新手且对相关代码无基本概念，我确实要直面“因果轮回”。

另外，对于多次修改仍无满意结果的部分，应该提醒 AI 重新思考、彻底思考，在我的经验范畴内，简单加上这么一句“温馨提示”居然是有效的。

总而言之，自然语言编程的效果是“可以理解的好”，而这已经使我足够满意了。
